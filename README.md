# QuizN Clone — 실시간 퀴즈 플랫폼

Node.js + Socket.IO 기반의 카훗/퀴즈앤 스타일 실시간 퀴즈 서비스입니다.
호스트는 **시작만 누르면 끝까지 자동 진행**됩니다.

## 실행 방법

```bash
npm install
npm start
# → http://localhost:3000
```

## 게임 흐름

```
[로비]  호스트가 PIN 공유, 참가자 모음
   ↓ 호스트가 "시작" 클릭 (이후 입력 불필요)
[문제]  타이머 카운트다운 (15초 등 설정값)
   ↓ 시간 종료 또는 전원 답변
[정답]  큰 글씨로 정답만 3초간 공개
   ↓ 자동 전환
[랭킹]  닉네임 + 점수를 점수 순으로 표시 (1등은 거대하게)
   ↓ 5초 후 자동 전환
[다음 문제] 또는 [최종 결과]
```

호스트 화면은 교실/회의실 빔프로젝터에 띄우는 용도로 디자인됐어요. 큰 글씨, 큰 타이머, 큰 랭킹.

## 사용법

1. **호스트** (큰 화면): `http://localhost:3000/host` → "방 만들기" → PIN 표시
2. **플레이어** (개인 폰): `http://localhost:3000/play` → PIN + 닉네임 입력
3. 호스트가 "퀴즈 시작" 한 번만 클릭 → 끝까지 자동 진행

## 프로젝트 구조

```
quizn-clone/
├── server.js          # 게임 상태 머신 (lobby→question→answer_reveal→ranking→ended)
├── quizzes.json       # 문제 데이터 — 여기만 고치면 컨텐츠 교체
├── public/
│   ├── index.html     # 랜딩
│   ├── host.html      # 호스트 화면 (큰 화면 + 자동 진행)
│   └── play.html      # 플레이어 화면 (모바일)
└── package.json
```

## 핵심 설계

### 자동 진행 (서버 주도 상태 머신)
서버가 `setTimeout`으로 모든 단계 전환을 관리. 클라이언트는 이벤트만 받아 표시:

| 이벤트 | 발생 시점 | 화면 |
|---|---|---|
| `question:show` | 문제 시작 | 4지선다 + 타이머 |
| `tick` | 매초 | 카운트다운 갱신 |
| `answer:reveal` | 시간종료/전원답변 후 자동 | 정답 큰 글씨 (3초) |
| `ranking:show` | 정답공개 3초 후 자동 | 점수 순 랭킹 (5초) |
| `question:show` 또는 `game:ended` | 랭킹 5초 후 자동 | 다음 문제 / 최종 |

조정 가능한 상수 (server.js 상단):
- `ANSWER_REVEAL_SEC = 3` — 정답 표시 시간
- `RANKING_SEC = 5` — 랭킹 표시 시간

### 정답 누설 방지
서버가 `question:show` 보낼 때 `answer` 필드는 뺍니다. 정답은 `answer:reveal` 시점에만 노출.

### 점수 계산 (Kahoot 방식)
정답: `500 + 500 × (남은시간/제한시간)` → 빨리 맞힐수록 1000점에 가까운 점수.

### 랭킹 시각적 강조
호스트 화면에서 점수 순서대로:
- 🥇 1등: 거대한 글씨 (42px) + 금색 그라데이션
- 🥈 2등: 큰 글씨 (34px) + 은색
- 🥉 3등: 중간 (28px) + 동색
- 4~6위: 보통 (22px)
- 7위 이하: 작게 (18px)

플레이어 화면에서는 **자기 닉네임이 강조**됩니다 (보라색 박스로).

## 문제 수정

`quizzes.json` 의 `default.questions` 배열 편집:

```json
{
  "q": "문제 텍스트",
  "choices": ["선택1", "선택2", "선택3", "선택4"],
  "answer": 0,         // 0~3 정답 인덱스
  "timeLimit": 15      // 초
}
```

## 사내/학교용 확장 아이디어

1. **결과 CSV/Excel export** — 게임 종료 시 자동 다운로드
2. **다중 퀴즈셋** — `/api/rooms` body에 `quizId` 받아서 선택
3. **DB 저장** — SQLite로 게임 결과 영속화 (memory → disk)
4. **SSO 로그인** — Google Workspace로 호스트 인증
5. **객관식 외 문제 유형** — OX, 주관식, 순서 맞추기
6. **Redis 어댑터** — 수백 명 동시 접속 시 멀티 인스턴스

## 알려진 한계

- 메모리 기반: 서버 재시작 = 진행 중인 모든 게임 초기화
- 호스트가 새로고침하면 방 사라짐 (재접속 토큰 미구현)
- 플레이어 끊김 시 점수 복원 안 됨 (socketId로만 식별)
- 동시 50~200명은 무리없음. 그 이상은 Redis 어댑터 + LB 권장
